> **Задача:**  
> Построить многостраничный отчёт для контроля продаж и рентабельности на маркетплейсе, объединив данные из Google Sheets и PostgreSQL (API Wildberries), и реализовать продвинутую навигацию для удобства использования менеджером.

## 🎯 Бизнес-контекст

Заказчику требовался полноценный аналитический инструмент, который позволит:
- видеть общую картину по выручке, прибыли, затрат на логистику, АКБ, рентабельности.
- моментально проваливаться в проблемные зоны (минусовая прибыль, низкая рентабельность товаров),
- управлять отчётом через **кнопки, закладки и фильтры**, как в полноценном приложении.

## 🔧 Что сделано

### Подключение и модель данных
- Интеграция с **Google Sheets** (справочники, планы) и **PostgreSQL** (транзакции, поступающие через API Wildberries).
- Построена модель «звезда» с таблицами фактов (продажи, затраты) и справочниками (товары, менеджеры, даты).
- Реализован **кастомный календарь на DAX**, так как исходный справочник дат был неполным.

### Сложная бизнес-логика (DAX)
- Расчёт **рентабельности** с учётом логистических затрат, себестоимости и комиссий.
- Динамический **ABC‑анализ** по выручке, прибыли и количеству заказов.
- Определение **заказов с минусовой прибылью** и автоматическая подсветка убыточных менеджеров.
- Сравнение среднего чека и рентабельности в разрезе сегментов покупателей (Consumer, Corporate, Home Office, Small Business).

### Интерактивность и навигация
- Реализованы **кнопки** для переключения между страницами отчёта без использования стандартных вкладок.
- Все визуальные элементы оснащены **кросс‑фильтрацией** .
- Фильтры по дате, сегменту, менеджеру и номенклатуре синхронизированы между всеми вкладками.

### Визуализация
- Карта распределения продаж и затрат на логистику.
- Динамика рентабельности помесячно с возможностью drill‑down до дня.
- Таблица убыточных менеджеров с мгновенной сортировкой по величине потерь.
- Гистограмма рентабельности по номенклатуре (от убыточных категорий до высокомаржинальных).
